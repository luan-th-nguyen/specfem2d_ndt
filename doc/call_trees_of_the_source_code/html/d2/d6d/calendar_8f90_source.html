94  integer function idaywk(jdayno)
95 
96 ! IDAYWK = compute the DAY of the WeeK given the Julian Day number,
97 ! version 1.0.
98 
99  implicit none
100 
101 ! Input variable
102  integer, intent(in) :: jdayno
103 ! jdayno = Julian Day number starting at noon of the day in question.
104 
105 ! Output of the function:
106 ! idaywk = day of the week, where 0=Sunday, 1=Monday, ..., 6=Saturday.
107 
108 !----------
109 ! Compute the day of the week given the Julian Day number.
110 ! You can find the Julian Day number given (day,month,year)
111 ! using subroutine calndr below.
112 ! Example: For the first day of the Gregorian calendar,
113 ! Friday 15 October 1582, compute the Julian day number (option 3 of
114 ! subroutine calndr) and compute the day of the week.
115 ! call calndr (3, 15, 10, 1582, jdayno)
116 ! write(*,*) jdayno, idaywk(jdayno)
117 ! The numbers printed should be 2299161 and 5, where 5 refers to Friday.

121 ! See www.gnu.org for details.

123 ! but WITHOUT ANY WARRANTY; without even the implied warranty of

125 ! If you find any errors, please notify:

128 ! Florida State University

137 ! jdSun is the Julian Day number starting at noon on any Sunday.
138 ! I arbitrarily chose the first Sunday after Julian Day 1,
139 ! which is Julian Day 6.
140  integer, parameter :: jdSun = 6
141 
142  idaywk = mod(jdayno-jdsun,7)
143

145 ! number. Fortran's built-in mod function returns a negative value
146 ! when the argument is negative. In that case, we adjust the result

156  subroutine calndr(iday,month,iyear,idayct)
157 
158 ! CALNDR = CALeNDaR conversions, version 1.0
159 
160  implicit none
161 
162 ! specify the desired calendar conversion option.
163 ! in order to return the julian day number, compatible with function idaywk from above,
164 ! we choose option 3
165 ! (tested with dates: Feb, 23 2010 -> idaywk = Tue
166 ! Dec, 24 2009 -> idaywk = Thu
167 ! Oct, 15 1582 -> idaywk = Fri ...which all look o.k. )
168  integer, parameter :: ioptn = 3
169 
170 ! Input/Output variables
171  integer, intent(inout) :: iday,month,iyear,idayct
172 
173 !----------
174 !
175 ! subroutine calndr() performs calendar calculations using either
176 ! the standard Gregorian calendar or the old Julian calendar.
177 ! This subroutine extends the definitions of these calendar systems
178 ! to any arbitrary year. The algorithms in this subroutine
179 ! will work with any date in the past or future,
180 ! but overflows will occur if the numbers are sufficiently large.
181 ! For a computer using a 32-bit integer, this routine can handle
182 ! any date between roughly 5.8 million BC and 5.8 million AD
183 ! without experiencing overflow during calculations.
184 !
185 ! No external functions or subroutines are called.
186 !
187 !----------
188 !
189 ! input/output arguments for subroutine CALNDR()
190 !
191 ! "ioptn" is the desired calendar conversion option explained below.
192 ! Positive option values use the standard modern Gregorian calendar.
193 ! Negative option values use the old Julian calendar which was the
194 ! standard in Europe from its institution by Julius Caesar in 45 BC
195 ! until at least 4 October 1582. The Gregorian and Julian calendars
196 ! are explained further below.
197 !
198 ! (iday,month,iyear) is a calendar date where "iday" is the day of
199 ! the month, "month" is 1 for January, 2 for February, etc.,
200 ! and "iyear" is the year. If the year is 1968 AD, enter iyear=1968,
201 ! since iyear=68 would refer to 68 AD.
202 ! For BC years, iyear should be negative, so 45 BC would be iyear=-45.
203 ! By convention, there is no year 0 under the BC/AD year numbering
204 ! scheme. That is, years proceed as 2 BC, 1 BC, 1 AD, 2 AD, etc.,
205 ! without including 0. The subroutine calndr() will print an error message
206 ! and stop if you specify iyear=0.
207 !
208 ! "idayct" is a day count. It is either the day number during the
209 ! specified year or the Julian Day number, depending on the value
210 ! of ioptn. By day number during the specified year, we mean
211 ! idayct=1 on 1 January, idayct=32 on 1 February, etc., to idayct=365
212 ! or 366 on 31 December, depending on whether the specified year
213 ! is a leap year.
214 !
215 ! The values of input variables are not changed by this subroutine.
216 !
217 !
218 ! ALLOWABLE VALUES FOR "IOPTN" and the conversions they invoke.
219 ! Positive option values ( 1 to 5) use the standard Gregorian calendar.
220 ! Negative option values (-1 to -5) use the old Julian calendar.
221 !
222 ! Absolute
223 ! value
224 ! of ioptn Input variable(s) Output variable(s)
225 !
226 ! 1 iday,month,iyear idayct
227 ! Given a calendar date (iday,month,iyear), compute the day number
228 ! (idayct) during the year, where 1 January is day number 1 and
229 ! 31 December is day number 365 or 366, depending on whether it is
230 ! a leap year.
231 !
232 ! 2 idayct,iyear iday,month
233 ! Given the day number of the year (idayct) and the year (iyear),
234 ! compute the day of the month (iday) and the month (month).
235 !
236 ! 3 iday,month,iyear idayct
237 ! Given a calendar date (iday,month,iyear), compute the Julian Day
238 ! number (idayct) that starts at noon of the calendar date specified.
239 !
240 ! 4 idayct iday,month,iyear
241 ! Given the Julian Day number (idayct) that starts at noon,
242 ! compute the corresponding calendar date (iday,month,iyear).
243 !
244 ! 5 idayct iday,month,iyear
245 ! Given the Julian Day number (idayct) that starts at noon,
246 ! compute the corresponding day number for the year (iday)
247 ! and year (iyear). On return from calndr(), "month" will always
248 ! be set equal to 1 when ioptn=5.
249 !
250 ! No inverse function is needed for ioptn=5 because it is
251 ! available through option 3. One simply calls calndr() with:
252 ! ioptn = 3,
253 ! iday = day number of the year instead of day of the month,
254 ! month = 1, and
255 ! iyear = whatever the desired year is.
256 !
257 !----------
258 !
259 ! EXAMPLES
260 ! The first 6 examples are for the standard Gregorian calendar.
261 ! All the examples deal with 15 October 1582, which was the first day
262 ! of the Gregorian calendar. 15 October is the 288-th day of the year.
263 ! Julian Day number 2299161 began at noon on 15 October 1582.
264 !
265 ! Find the day number during the year on 15 October 1582
266 ! ioptn = 1
267 ! call calndr (ioptn, 15, 10, 1582, idayct)
268 ! calndr() should return idayct=288
269 !
270 ! Find the day of the month and month for day 288 in year 1582.
271 ! ioptn = 2
272 ! call calndr (ioptn, iday, month, 1582, 288)
273 ! calndr() should return iday=15 and month=10.
274 !
275 ! Find the Julian Day number for 15 October 1582.
276 ! ioptn = 3
277 ! call calndr (ioptn, 15, 10, 1582, julian)
278 ! calndr() should return julian=2299161
279 !
280 ! Find the Julian Day number for day 288 during 1582 AD.
281 ! When the input is day number of the year, one should specify month=1
282 ! ioptn = 3
283 ! call calndr (ioptn, 288, 1, 1582, julian)
284 ! calndr() should return dayct=2299161
285 !
286 ! Find the date for Julian Day number 2299161.
287 ! ioptn = 4
288 ! call calndr (ioptn, iday, month, iyear, 2299161)
289 ! calndr() should return iday=15, month=10, and iyear=1582
290 !
291 ! Find the day number during the year (iday) and year
292 ! for Julian Day number 2299161.
293 ! ioptn = 5
294 ! call calndr (ioptn, iday, month, iyear, 2299161)
295 ! calndr() should return iday=288, month= 1, iyear=1582
296 !
297 ! Given 15 October 1582 under the Gregorian calendar,
298 ! find the date (idayJ,imonthJ,iyearJ) under the Julian calendar.
299 ! To do this, we call calndr() twice, using the Julian Day number
300 ! as the intermediate value.
301 ! call calndr ( 3, 15, 10, 1582, julian)
302 ! call calndr (-4, idayJ, monthJ, iyearJ, julian)
303 ! The first call to calndr() should return julian=2299161, and
304 ! the second should return idayJ=5, monthJ=10, iyearJ=1582
305 !
306 !----------
307 !
308 ! BASIC CALENDAR INFORMATION
309 !
310 ! The Julian calendar was instituted by Julius Caesar in 45 BC.
311 ! Every fourth year is a leap year in which February has 29 days.
312 ! That is, the Julian calendar assumes that the year is exactly
313 ! 365.25 days long. Actually, the year is not quite this long.
314 ! The modern Gregorian calendar remedies this by omitting leap years
315 ! in years divisible by 100 except when the year is divisible by 400.
316 ! Thus, 1700, 1800, and 1900 are leap years under the Julian calendar
317 ! but not under the Gregorian calendar. The years 1600 and 2000 are
318 ! leap years under both the Julian and the Gregorian calendars.
319 ! Other years divisible by 4 are leap years under both calendars,
320 ! such as 1992, 1996, 2004, 2008, 2012, etc. For BC years, we recall
321 ! that year 0 was omitted, so 1 BC, 5 BC, 9 BC, 13 BC, etc., and 401 BC,
322 ! 801 BC, 1201 BC, etc., are leap years under both calendars, while
323 ! 101 BC, 201 BC, 301 BC, 501 BC, 601 BC, 701 BC, 901 BC, 1001 BC,
324 ! 1101 BC, etc., are leap years under the Julian calendar but not
325 ! the Gregorian calendar.
326 !
327 ! The Gregorian calendar is named after Pope Gregory XIII. He declared
328 ! that the last day of the old Julian calendar would be Thursday,
329 ! 4 October 1582 and that the following day, Friday, would be reckoned
330 ! under the new calendar as 15 October 1582. The jump of 10 days was
331 ! included to make 21 March closer to the spring equinox.
332 !
333 ! Only a few Catholic countries (Italy, Poland, Portugal, and Spain)
334 ! switched to the Gregorian calendar on the day after 4 October 1582.
335 ! It took other countries months to centuries to change to the
336 ! Gregorian calendar. For example, England's first day under the
337 ! Gregorian calendar was 14 September 1752. The same date applied to
338 ! the entire British empire, including America. Japan, Russia, and many
339 ! eastern European countries did not change to the Gregorian calendar
340 ! until the 20th century. The last country to change was Turkey,
341 ! which began using the Gregorian calendar on 1 January 1927.
342 !
343 ! Therefore, between the years 1582 and 1926 AD, you must know
344 ! the country in which an event was dated to interpret the date
345 ! correctly. In Sweden, there was even a year (1712) when February
346 ! had 30 days. Consult a book on calendars for more details
347 ! about when various countries changed their calendars.
348 !
349 ! DAY NUMBER DURING THE YEAR
350 ! The day number during the year is simply a counter equal to 1 on
351 ! 1 January, 32 on 1 February, etc., thorugh 365 or 366 on 31 December,
352 ! depending on whether the year is a leap year. Sometimes this is
353 ! called the Julian Day, but that term is better reserved for the
354 ! day counter explained below.
355 !
356 ! JULIAN DAY NUMBER
357 ! The Julian Day numbering system was designed by Joseph Scaliger
358 ! in 1582 to remove ambiguity caused by varying calendar systems.
359 ! The name "Julian Day" was chosen to honor Scaliger's father,
360 ! Julius Caesar Scaliger (1484-1558), an Italian scholar and physician
361 ! who lived in France. Because Julian Day numbering was especially
362 ! designed for astronomers, Julian Days begin at noon so that the day
363 ! counter does not change in the middle of an astronmer's observing
364 ! period. Julian Day 0 began at noon on 1 January 4713 BC under the
365 ! Julian calendar. A modern reference point is that 23 May 1968
366 ! (Gregorian calendar) was Julian Day 2,440,000.
367 !
368 ! JULIAN DAY NUMBER EXAMPLES
369 !
370 ! The table below shows a few Julian Day numbers and their corresponding
371 ! dates, depending on which calendar is used. A negative 'iyear' refers
372 ! to BC (Before Christ).
373 !
374 ! Julian Day under calendar:
375 ! iday month iyear Gregorian Julian
376 ! 24 11 -4714 0 -38
377 ! 1 1 -4713 38 0
378 ! 1 1 1 1721426 1721424
379 ! 4 10 1582 2299150 2299160
380 ! 15 10 1582 2299161 2299171
381 ! 1 3 1600 2305508 2305518
382 ! 23 5 1968 2440000 2440013
383 ! 5 7 1998 2451000 2451013
384 ! 1 3 2000 2451605 2451618
385 ! 1 1 2001 2451911 2451924
386 !
387 ! From this table, we can see that the 10 day difference between the
388 ! two calendars in 1582 grew to 13 days by 1 March 1900, since 1900 was
389 ! a leap year under the Julian calendar but not under the Gregorian
390 ! calendar. The gap will widen to 14 days after 1 March 2100 for the
391 ! same reason.
392 !
393 !----------
394 !
395 ! PORTABILITY
396 !
397 ! This subroutine is written in standard FORTRAN 90.
398 ! It calls no external functions or subroutines and should run
399 ! without problem on any computer having a 32-bit word or longer.
400 !
401 !----------
402 !
403 ! ALGORITHM
404 !
405 ! The goal in coding calndr() was clear, clean code, not efficiency.
406 ! Calendar calculations usually take a trivial fraction of the time
407 ! in any program in which dates conversions are involved.
408 ! Data analysis usually takes the most time.
409 !
410 ! Standard algorithms are followed in this subroutine. Internal to
411 ! this subroutine, we use a year counter "jyear" such that
412 ! jyear=iyear when iyear is positive
413 ! =iyear+1 when iyear is negative.
414 ! Thus, jyear does not experience a 1 year jump like iyear does
415 ! when going from BC to AD. Specifically, jyear=0 when iyear=-1,
416 ! i.e., when the year is 1 BC.
417 !
418 ! For simplicity in dealing with February, inside this subroutine,
419 ! we let the year begin on 1 March so that the adjustable month,
420 ! February is the last month of the year.
421 ! It is clear that the calendar used to work this way because the
422 ! months September, October, November, and December refer to
423 ! 7, 8, 9, and 10. For consistency, jyear is incremented on 1 March
424 ! rather than on 1 January. Of course, everything is adjusted back to
425 ! standard practice of years beginning on 1 January before answers
426 ! are returned to the routine that calls calndr().
427 !
428 ! Lastly, we use a trick to calculate the number of days from 1 March
429 ! until the end of the month that precedes the specified month.
430 ! That number of days is int(30.6001*(month+1))-122,
431 ! where 30.6001 is used to avoid the possibility of round-off and
432 ! truncation error. For example, if 30.6 were used instead,
433 ! 30.6*5 should be 153, but round-off error could make it 152.99999,
434 ! which would then truncated to 152, causing an error of 1 day.
435 !
436 ! Algorithm reference:
437 ! Dershowitz, Nachum and Edward M. Reingold, 1990: Calendrical
438 ! Calculations. Software-Practice and Experience, vol. 20, number 9
439 ! (September 1990), pp. 899-928.
440 !

443 ! See www.gnu.org for details.

445 ! but WITHOUT ANY WARRANTY; without even the implied warranty of
446 ! MERCHANTABILITY or FITNESS FOR A PARTICULAR PURPOSE.

459 ! Declare internal variables.
460  integer jdref, jmonth, jyear, leap, n1yr, n4yr, n100yr, n400yr, ndays, ndy400, ndy100, nyrs, yr400, yrref
461 !
462 ! Explanation of all internal variables.
463 ! jdref Julian Day on which 1 March begins in the reference year.
464 ! jmonth Month counter which equals month+1 if month > 2
465 ! or month+13 if month <= 2.
466 ! jyear Year index, jyear=iyear if iyear > 0, jyear=iyear+1
467 ! if iyear < 0. Thus, jyear does not skip year 0
468 ! like iyear does between BC and AD years.
469 ! leap =1 if the year is a leap year, =0 if not.
470 ! n1yr Number of complete individual years between iyear and
471 ! the reference year after all 4, 100,
472 ! and 400 year periods have been removed.
473 ! n4yr Number of complete 4 year cycles between iyear and
474 ! the reference year after all 100 and 400 year periods
475 ! have been removed.
476 ! n100yr Number of complete 100 year periods between iyear and
477 ! the reference year after all 400 year periods
478 ! have been removed.
479 ! n400yr Number of complete 400 year periods between iyear and
480 ! the reference year.
481 ! ndays Number of days since 1 March during iyear. (In intermediate
482 ! steps, it holds other day counts as well.)
483 ! ndy400 Number of days in 400 years. Under the Gregorian calendar,
484 ! this is 400*365 + 100 - 3 = 146097. Under the Julian
485 ! calendar, this is 400*365 + 100 = 146100.
486 ! ndy100 Number of days in 100 years, Under the Gregorian calendar,
487 ! this is 100*365 + 24 = 36524. Under the Julian calendar,
488 ! this is 100*365 + 25 = 36525.
489 ! nyrs Number of years from the beginning of yr400
490 ! to the beginning of jyear. (Used for option +/-3).
491 ! yr400 The largest multiple of 400 years that is <= jyear.
492 !
493 !
494 !----------------------------------------------------------------
495 ! Do preparation work.
496 !
497 ! Look for out-of-range option values.
498  if ((ioptn == 0) .or. (abs(ioptn) >= 6)) then
499  write(*,*)'For calndr(), you specified ioptn = ', ioptn
500  write(*,*) 'Allowable values are 1 to 5 for the Gregorian calendar'
501  write(*,*) 'and -1 to -5 for the Julian calendar.'
502  stop
503  endif
504 !
505 ! Options 1-3 have "iyear" as an input value.
506 ! Internally, we use variable "jyear" that does not have a jump
507 ! from -1 (for 1 BC) to +1 (for 1 AD).
508  if (abs(ioptn) <= 3) then
509  if (iyear > 0) then
510  jyear = iyear
511  else if (iyear == 0) then
512  write(*,*) 'For calndr(), you specified the nonexistent year 0'
513  stop
514  else
515  jyear = iyear + 1
516  endif
517 !
518 ! Set "leap" equal to 0 if "jyear" is not a leap year
519 ! and equal to 1 if it is a leap year.
520  leap = 0
521  if ((jyear/4)*4 == jyear) then
522  leap = 1
523  endif
524  if ((ioptn > 0) .and. &
525  ((jyear/100)*100 == jyear) .and. &
526  ((jyear/400)*400 /= jyear) ) then
527  leap = 0
528  endif
529  endif
530 !
531 ! Options 3-5 involve Julian Day numbers, which need a reference year
532 ! and the Julian Days that began at noon on 1 March of the reference
533 ! year under the Gregorian and Julian calendars. Any year for which
534 ! "jyear" is divisible by 400 can be used as a reference year.
535 ! We chose 1600 AD as the reference year because it is the closest
536 ! multiple of 400 to the institution of the Gregorian calendar, making
537 ! it relatively easy to compute the Julian Day for 1 March 1600
538 ! given that, on 15 October 1582 under the Gregorian calendar,
539 ! the Julian Day was 2299161. Similarly, we need to do the same
540 ! calculation for the Julian calendar. We can compute this Julian
541 ! Day knwoing that on 4 October 1582 under the Julian calendar,
542 ! the Julian Day number was 2299160. The details of these calculations
543 ! is next.
544 ! From 15 October until 1 March, the number of days is the remainder
545 ! of October plus the days in November, December, January, and February:
546 ! 17+30+31+31+28 = 137, so 1 March 1583 under the Gregorian calendar
547 ! was Julian Day 2,299,298. Because of the 10 day jump ahead at the
548 ! switch from the Julian calendar to the Gregorian calendar, 1 March
549 ! 1583 under the Julian calendar was Julian Day 2,299,308. Making use
550 ! of the rules for the two calendar systems, 1 March 1600 was Julian
551 ! Day 2,299,298 + (1600-1583)*365 + 5 (due to leap years) =
552 ! 2,305,508 under the Gregorian calendar and day 2,305,518 under the
553 ! Julian calendar.
554 ! We also set the number of days in 400 years and 100 years.
555 ! For reference, 400 years is 146097 days under the Gregorian calendar
556 ! and 146100 days under the Julian calendar. 100 years is 36524 days
557 ! under the Gregorian calendar and 36525 days under the Julian calendar.
558  if (abs(ioptn) >= 3) then
559 !
560 ! Julian calendar values.
561  yrref = 1600
562  jdref = 2305518
563 ! = Julian Day reference value for the day that begins
564 ! at noon on 1 March of the reference year "yrref".
565  ndy400 = 400*365 + 100
566  ndy100 = 100*365 + 25
567 !
568 ! Adjust for Gregorian calendar values.
569  if (ioptn > 0) then
570  jdref = jdref - 10
571  ndy400 = ndy400 - 3
572  ndy100 = ndy100 - 1
573  endif
574  endif
575 !
576 !----------------------------------------------------------------
577 ! OPTIONS -1 and +1:
578 ! Given a calendar date (iday,month,iyear), compute the day number
579 ! of the year (idayct), where 1 January is day number 1 and 31 December
580 ! is day number 365 or 366, depending on whether it is a leap year.
581  if (abs(ioptn) == 1) then
582 !
583 ! Compute the day number during the year.
584  if (month <= 2) then
585  idayct = iday + (month-1)*31
586  else

699 ! where ndays=0 on 1 March.
700 !

702 ! = year, as counted in the standard way,
703 ! but relative to 1 March.
704 !
705 ! At this point, we need to separate ioptn=abs(4), which seeks a
706 ! calendar date, and ioptn=abs(5), which seeks the day number during
707 ! the year. First compute the calendar date if desired (abs(ioptn)=4).

709  jmonth = (10*(ndays+31))/306 + 3

711 ! December, =14 for January, =15 for February.
712  iday = (ndays+123) - int(30.6001*jmonth)

714 ! of the month.
715 !

717 ! on 1 January.
718  if (jmonth <= 13) then
719  month = jmonth - 1
720  else
721  month = jmonth - 13
722  iyear = iyear + 1
723  endif
724 !
725 ! This code handles abs(ioptn)=5, finding the day number during the year.
726  else
727 ! ioptn=5 always returns month= 1, which we set now.
728  month = 1

730 ! We need to determine whether this is a leap year.
731  leap = 0
732  if ((jyear/4)*4 == jyear) then
733  leap = 1
734  endif
735  if ((ioptn > 0) .and. &
736  ((jyear/100)*100 == jyear) .and. &
737  ((jyear/400)*400 /= jyear) ) then
738  leap = 0
739  endif

742 ! ndays is the number of days since the most recent 1 March,
743 ! so ndays=0 on 1 March.
744  if (ndays <= 305) then
745  iday = ndays + 60 + leap
746  else
747  iday = ndays - 305
748  iyear = iyear + 1
749  endif
750  endif
751 !
752 ! Adjust the year if it is <= 0, and hence BC (Before Christ).
753  if (iyear <= 0) then
754  iyear = iyear - 1
755  endif
756 !
757 ! End the code for the last option, ioptn.
758  endif
