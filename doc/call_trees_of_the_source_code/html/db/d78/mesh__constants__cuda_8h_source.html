36 /* trivia
37 
38 - for most working arrays we use now "realw" instead of "float" type declarations to make it easier to switch
39  between a real or double precision simulation
40  (matching CUSTOM_REAL == 4 or 8 in fortran routines).
41 
42 - instead of boolean "logical" declared in fortran routines, in C (or Cuda-C) we have to use "int" variables.
43  ifort / gfortran caveat:
44  to check whether it is true or false, do not check for == 1 to test for true values since ifort just uses
45  non-zero values for true (e.g. can be -1 for true). however, false will be always == 0.
46  thus, rather use: if (var) {...} for testing if true instead of if (var == 1){...} (alternative: one could use if (var != 0) {...}
47 
48 */
